OVERVIEW SPECIFICATION

VISUAL SEARCH

ABOUT

Provide a URL to find products that are similar to any clothing items found in the image. Obtain full product names, IDs, prices and discounts, and direct links to the corresponding brand's web store.

MAIN GOAL

With this API, you can:

  • Provide an image of a product you have interest in via a URL, and get similar products currently available in Zara.
  • Get product IDs and names, the current price and also the original price if there's a discount, and a link to the product's web store, and the brand behind the product.
  • Paginate through results to easily manage large data sets.

CONTEXT

This API is available through the Inditex Developer Portal. To be able to use it, you first need to sign up, create an application, and then add this API to it.

CONFIGURATION

This API does not require any particular configuration.

SPECIFICS

AUTHENTICATION

You can use this API in your service in sandbox and production environments with OAuth2 authentication. See Promote to production guide to know how to authenticate in both environments and start making requests right away.

RATE LIMIT

This API, along with all the Developer Portal APIs, has a rate limit that you can check in the information button in the API card. In case this limit is exceeded, you will get a 429 HTTP error code.

USAGE

If you want to take a peek at this API via web, you can do it accessing the Specification tab. Check the Promote to production guide to learn how to use the Try it out functionality.

GET STARTED WITH A USE CASE

Wouldn't it be useful to find a product similar to one you see someone wearing in the street? This API can be used for that purpose, among others. You could make an application that enables users to take a picture and give them the Zara link so they can directly buy the products.

To make this application work, you can start by registering, logging in, and creating an application in the Developer Portal that contains the Visual Search API. Once you have that, you can start making sandbox calls to integrate with this API, and when you already have integrated, you can promote your application to production to start getting real data.

MAKE A SANDBOX CALL

  1. Request a JWT to the auth.inditex.com sandbox realm specifying your credentials along with the scopes of the APIs you want to access so that you can make requests to the APIs in your application.

See how to do this with curl in the Promote to production guide.

  1. Make a CURL with the token and the base URL of the API that is displayed in the specification tab of the Visual Search API Detail page.

To provide the user's image, you can use the GET /products operation like this:

curl -X GET "https://api-sandbox.inditex.com/pubvsearch-sandbox/products?image=YOUR_IMAGE_URL" \
    -H "Authorization: Bearer YOUR_JWT_TOKEN" \
    -H "Content-Type: application/json"
  • Replace YOUR_IMAGE_URL with the URL that leads to an image that is under 7 MB in size.
  • Replace YOUR_JWT_TOKEN with your actual JWT code obtained in step 1.

This will retrieve you with the similar products' ID, their name, their current price and also the original price range for products with an applied discount, links to the products Web store, and the brand behind the product.

MAKE A PRODUCTION CALL

Whenever you want to switch to the production environment, repeat the same steps, but note these differences:

  • You need to promote your application to production.
  • The JWT URL: Obtain the production JWT from the production realm.
  • The API Base URL: Use the production API base URL (provided in the API detail page).

ENDPOINTS

VISUALLY SEARCH PRODUCTS

PARAMETER TYPE DESCRIPTION REQUIRED EXAMPLE
image string URL that leads to an image that is under 7 MB in size Yes https://es.pinterest.com/pin/622411611023151390/
page int Page number of the result (default:1) No 1
perPage int Number of products per page (default: 5) No 5

Example request

Obtain a list of jeans similar to ones seen in Pinterest.

curl -X GET "https://api-sandbox.inditex.com/pubvsearch-sandbox/products?image=https://es.pinterest.com/pin/622411611023151390/" \
    -H "Authorization: Bearer YOUR_JWT_TOKEN" \
    -H "Content-Type: application/json"

Response structure

FIELD TYPE DESCRIPTION EXAMPLE
id string Unique identifier for the product 367196402
name string Product name METALLIC THREAD RUSTIC SHIRT
price.currency string The currency code in ISO 4217 format. Possible value: EUR EUR
price.value.current float The current selling price of the product 15.99
price.value.original float The original price before any discount. Whenever a product has a discount, the price object returns the original attribute. If there's no discount, the original attribute isn't returned. 27.95
link string Product URL on the website of the corresponding brand. Link formats vary from brand to brand, so links can and will change accordingly. https://www.zara.com/es/en/metallic-thread-rustic-shirt-p02298151.html?v1=367196402
brand string Brand of the product zara

ERROR HANDLING

Here's what you might encounter if something goes wrong:

STATUS CODE DESCRIPTION SOLUTION
200 Successful request Successful response with product search results. -
400 Bad Request Invalid request parameters. Modify the request so that the request parameters are correct.
401 Unauthorized Request User not authorized or authentication invalid. Provide the correct credentials to authenticate.
403 Forbidden User hasn't got permissions to perform the request. -
404 No products found No products were found matching the given criteria. Try another query parameters.
429 Too many requests You have exceeded the maximum number of requests allowed. Limit your API calls to the API rate limit.
500 Internal server error An unexpected error has occurred on the server. If the problem persists, please contact us at supportapi@inditex.com.
